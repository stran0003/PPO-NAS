🔬 基于PPO的神经架构搜索（NAS）研究方案

面向数字预失真（DPD）七层混合模型的软参搜索与量化位宽分配

汇报日期：2026年6月1日

一、研究背景与动机

当前已设计完成一个七层混合结构基线模型用于数字预失真（DPD）任务。该模型结合了线性卷积层与LUT（Look-Up Table）非线性层。在实际部署中，模型的参数量、计算资源和存储资源均受到严格约束，因此需要在保证性能的前提下，自动搜索最优的架构超参数配置。

本研究拟采用PPO（Proximal Policy Optimization）强化学习算法作为NAS的搜索策略，分两个阶段分别解决"软参搜索"和"量化位宽分配"两个核心问题。

  • 1×1 通道全连接卷积：用 1×1 卷积核实现通道间的全连接混合

三、搜索空间定义

第一阶段 软参搜索空间

目标：在保证模型性能的前提下，尽可能降低资源量（参数量、计算量），寻找性能与资源的 Pareto 最优。

搜索维度适用层候选值 / 范围说明
卷积核大小 线性层（L1–L3, L5–L7） 如 {3, 5, 7, 9, 11} 更大的核捕获更长的时域依赖，但增加计算量
卷积类型 线性层（L1–L3, L5–L7） {分组卷积, 1×1 通道全连接卷积} 分组卷积参数少；1×1全连接表达能力强
样条个数 LUT 非线性层（L4） 如 {4, 8, 16, 32, 64} 样条数决定非线性拟合能力与存储开销的权衡
初始化策略 所有层 卷积层：{中心抽头=1/其余=0, Xavier, Kaiming}
LUT层：{1e-4 小值初始化, 线性初始化}		 初始化影响收敛速度和最终性能
归一化策略 数据预处理 {除以 2^15 固定位宽归一化, 其他归一化方案} 输入数据和 label 数据的归一化方式影响训练稳定性

第二阶段 量化位宽分配

目标：在软参搜索确定的架构基础上，进一步搜索每层的最优量化位宽，降低存储资源需求。此阶段暂不需要细化，先建立框架。

搜索维度适用层候选值说明
权重位宽所有层如 {4bit, 6bit, 8bit, 16bit}逐层独立搜索，关键层用高位宽，非关键层用低位宽
激活值位宽所有层如 {8bit, 16bit}与权重位宽联合搜索
LUT 表项位宽LUT 非线性层如 {8bit, 12bit, 16bit}LUT 层特有的存储优化维度

四、PPO 搜索框架设计

4.1 为什么选择 PPO

  • 稳定性好：PPO 通过 clip 机制限制策略更新幅度，避免策略崩溃，这对搜索空间离散且评估代价高的 NAS 场景至关重要
  • 样本效率较高：相比 REINFORCE，PPO 支持多轮小批量更新，能更充分利用每次采样的架构评估结果
  • 易于处理多目标：可以灵活地将多目标奖励信号融入策略梯度更新
  • 文献支持：MARCO（2024）等工作已验证 PPO 在 NAS + 量化联合搜索中的有效性

4.2 MDP 建模

MDP 要素定义
状态 s_t 当前已生成的架构部分（已决策层数、各层超参数），可用 RNN/LSTM 编码
动作 a_t 为当前层选择超参数：卷积核大小、卷积类型、样条数、初始化策略、归一化策略（第一阶段）；各层量化位宽（第二阶段）
策略 π_θ(a_t|s_t) 由控制器 RNN/LSTM 输出动作概率分布，逐层自回归生成完整架构（Actor 网络）
价值 V_φ(s_t) 由 Critic 网络估计状态 s_t 的期望回报，用于计算优势函数 A_t = G_t − V(s_t)，减小策略梯度方差
奖励 R 在完整架构生成后一次性给出（延迟奖励），基于验证集性能和资源指标的加权组合

4.3 控制器（Actor）架构

建议采用 LSTM 控制器：每时间步输入前一层的决策嵌入向量，输出当前层的动作概率分布。对于七层模型，一次完整的 rollout 生成 7 个时间步的决策序列。

备选方案：Transformer 控制器，微软研究院（2024）报告比 RNN 控制器提升约 22% 的样本效率。

4.4 Critic（价值网络）设计 ★ 补充

4.4.1 Critic 的作用

PPO 是 Actor-Critic 算法，需要 Critic 网络估计每个状态的价值 V(s)，用于计算优势函数：

A_t = (r_t + γ·V(s_{t+1}) + γ²·V(s_{t+2}) + ...) − V(s_t) ≈ GAE(δ_t, δ_{t+1}, ...)

即使中间步骤的即时奖励 r_t = 0，Critic 也能通过估计"当前 partial 架构未来能得多少分"来为每一步产生独立的优势信号，从而告诉 Actor 每一步的决策是好是坏。

4.4.2 Critic 架构方案对比

方案结构优点缺点推荐度
方案A：共享骨干 + 独立头 同一个 LSTM 编码状态 → Actor 头（Softmax）和 Critic 头（Scalar）并行 参数高效；特征共享减少过拟合；训练最快 Actor 和 Critic 的梯度可能在共享层冲突 ⭐⭐⭐⭐⭐

4.4.5 Critic 损失函数

Critic 的训练目标是最小化其预测值与实际回报之间的均方误差（MSE）：

L_critic(φ) = (1/T) · Σ_t ( V_φ(s_t) − G_t )²

其中 G_t 是通过 GAE 从终端奖励反传得到的 return（即 advantage + V_φ(s_t) 的 detached 值）。

PPO 的总损失为 Actor 损失和 Critic 损失的加权和：

L_total = L_actor + c_v · L_critic − c_e · H(π)

  • c_v：Critic 损失系数，典型值 0.5～1.0
  • c_e：熵正则化系数（鼓励探索），典型值 0.01～0.1
  • H(π)：策略熵，防止过早收敛到次优架构

4.4.6 实现注意事项

  • 价值裁剪（Value Clipping）：与 PPO 的策略裁剪类似，可对 Critic 的更新幅度进行裁剪，防止单步价值估计突变破坏训练：
    V_clipped = V_old + clip(V_new − V_old, −ε, +ε)
    L_critic = max( (V_new − G)², (V_clipped − G)² )
  • 价值归一化：对 returns G_t 做 batch 归一化或维护运行均值/方差，使 Critic 的目标分布稳定，加速收敛
  • 梯度流控制：在计算 Actor 损失时，应 detach() Critic 的输出，防止 Critic 的梯度污染 Actor 的更新方向（反之亦然，需仔细设计反向传播路径）
  • 独立优化器 vs 共享优化器：推荐 Actor 和 Critic 使用同一个优化器（Adam），共享 LSTM 参数时自然联合优化；若采用方案 B（独立网络），则各用独立优化器

五、两个已知困难及初步思考

⚠️ 困难一：奖励函数设计

问题描述：奖励函数的设计直接关系到 PPO 能否收敛、能否找到真正好的架构。单一的性能指标（如最终 NMSE/EVM）可能无法全面反映架构质量；而多目标加权又面临权重调优困难，且加权求和无法到达非凸 Pareto 前沿上的解。

影响：如果奖励信号噪声过大或设计不当，策略梯度可能无法引导搜索走向高质量区域。

⚠️ 困难二：搜索时间过长

问题描述：每个 case 的数据完整训练（1000 epochs）需要约 20 分钟。且存在多个不同 case 的数据，单个 case 的表现不足以反映模型的泛化能力，需要在多个 case 上均表现良好才能说明搜索到的架构具有鲁棒性。假设每次评估需要训练 3–5 个 case，单次架构评估就可能需要 1–1.5 小时。

影响：PPO 需要大量采样和评估迭代，按完整训练计算，搜索时间可能达到天级别甚至更长。

初步缓解思路

💡 思路A：短训练代理（Early-Stopping Proxy）

将完整训练（1000 epochs）缩短为 300 epochs，用初步性能作为最终性能的代理指标。文献表明，NAS-Bench-201 上仅需 10–12 个 epoch 即可使排名相关性达到 0.6+；300 epochs 足以提供可靠的相对排序信号。

可结合 TSE（Training Speed Estimation）：用近期训练 loss 的累积和作为性能预测代理，在小预算下排名相关性优于零成本代理。

💡 思路B：权重共享（Weight Sharing）

每次与旧策略比较时，对于架构未改变的那些层，直接复用其已训练的权重，仅训练发生了变化的层。这类似于 ENAS 的参数共享机制，可以显著加速收敛。但需注意"优化差距"（optimization gap）问题——共享权重可能无法精确反映独立训练的性能。

六、实验设计概要

6.1 第一阶段实验（软参搜索）

  1. 基准测试：在多个典型 case 上评估当前人工设计的基线模型，记录性能（NMSE/EVM）和资源量（参数量、MACs）
  2. 搜索空间验证：随机采样若干架构并完整训练，验证搜索空间的合理性和性能分布范围
  3. PPO 搜索运行：配置奖励函数权重，运行 PPO-NAS 搜索（预计 100–500 次架构评估）
  4. Pareto 分析：在性能-资源平面上绘制 Pareto 前沿，选取 3–5 个代表性架构进行完整训练验证
  5. 跨 case 验证：将搜索到的最优架构在多个未参与搜索的 case 上测试泛化能力

6.2 第二阶段实验（量化位宽分配）

  1. 在第一阶段选定的最优架构基础上，定义逐层量化位宽搜索空间
  2. 引入量化感知训练（QAT）评估候选量化策略
  3. PPO 搜索最优位宽分配方案
  4. 与均匀量化基线（如全部 8bit / 全部 16bit）对比

6.3 多 Case 评估策略

在搜索过程中，每个候选架构在随机采样的 K 个 case（K=3~5）上快速训练（300 epochs），以平均性能作为奖励信号。这比单 case 评估更具泛化性，同时通过短训练控制时间开销。

八、预期贡献与创新点

  1. 首次将 PPO-NAS 应用于 DPD 混合模型：将强化学习驱动的架构搜索引入数字预失真领域，填补 NAS 在 DPD 中应用的研究空白
  2. 多维度统一搜索框架：在同一 PPO 框架下联合搜索卷积核大小、卷积类型、样条数、初始化策略和归一化策略，而非单独优化每个维度
  3. 两阶段渐进搜索：先搜索软参确定架构，再搜索量化位宽降低存储，形成从"粗粒度架构搜索"到"细粒度量化优化"的完整流水线
  4. 面向实际部署的资源感知搜索：以参数量和存储资源为直接优化目标，而非仅以浮点性能为指标，搜索到的架构更具实际部署价值
