📊 奖励函数设计方案与困难解决方案

基于文献调研的 PPO-NAS 奖励函数设计与搜索加速策略

汇报日期：2026年6月1日

执行摘要

本报告针对"基于 PPO 的 NAS 研究方案"中识别出的两大核心困难——奖励函数设计与搜索时间过长——进行了系统的文献调研。基于 MnasNet、NPG-NAS、ENAS、MARCO、APQ、RANK-NOSH 等代表性工作，提出了一套分层的奖励函数设计方案（包含四种可选策略）和三种搜索加速方案（短训练代理 + 渐进淘汰、权重共享超网、零成本代理预筛选）。

核心建议：奖励函数采用 NPG-NAS 风格的退火期望函数（ADF） + 多 case 平均性能的组合方案；搜索加速采用 短训练（300 epoch）+ RANK-NOSH 渐进淘汰为主策略，逐层权重共享为辅助策略。

第一部分：奖励函数设计方案

1.1 问题分析

DPD 模型的奖励函数需同时考虑以下维度：

  • 性能维度：NMSE（归一化均方误差）、EVM（误差向量幅度）、ACLR（邻道泄漏比）
  • 资源维度：参数量、MACs（乘加运算数）、推理延迟
  • 泛化维度：多个 case 上的性能均值和方差

关键挑战在于：这些目标之间存在竞争关系（更小的模型通常性能更差），且单一加权公式无法到达非凸 Pareto 前沿上的解（Chen et al., 2020）。

1.2 方案一：加权乘积奖励（MnasNet 风格）推荐作基线

来源：Tan et al., "MnasNet: Platform-Aware Neural Architecture Search for Mobile", CVPR 2019

核心思想：将多目标通过加权乘积组合为单一标量奖励。

R(m) = ACC(m) × [Cost(m) / T]w

其中：
  ACC(m) = 模型 m 的性能指标（如 -NMSE，越大越好）
  Cost(m) = 资源消耗（参数量或 MACs）
  T = 目标资源阈值
  w = 控制性能-资源权衡的权重（w < 0 时惩罚超出目标；w = -0.07 是原论文常用值）

针对 DPD 场景的适配：

R(arch) = (-NMSEavg) × [Params(arch) / Paramstarget]w × Penalty(EVM)

其中：
  NMSEavg = K 个 case 上的平均 NMSE（dB），取负号使"越小越好"变为"越大越好"
  Paramstarget = 目标参数量（如基线模型的 70%）
  w = -0.07 ~ -0.15，控制对超额参数的惩罚力度
  Penalty(EVM) = 若 EVM 超过门限则乘以 <1 的惩罚因子（可选）

⚠️ 局限性

每个权重配置只能找到 Pareto 前沿上的一个点；对于非凸 Pareto 前沿，该方法可能完全错过某些高质量解。TabNAS（2022）发现 MnasNet 奖励在约束搜索空间中仅有 <1% 的成功率。

1.3 方案二：退火期望函数（ADF, NPG-NAS 风格）强烈推荐

来源：Chen et al., "Multi-objective Neural Architecture Search via Non-stationary Policy Gradient", arXiv:2001.08437

核心思想：不固定资源目标的权重，而是让资源目标在整个搜索过程中逐步退火，使策略能够适应从"宽松"到"严格"的资源约束，在单次搜索运行中覆盖整个 Pareto 前沿。

核心机制：三角期望函数 F2(τ, δ)

设资源消耗 c = Params(arch)，定义期望函数：
  F2(c; τ, δ) = max(0, 1 - |c - τ| / δ)

其中 τ 是当前目标资源值（退火参数），δ 是容差宽度。

退火调度：
  第 t 次迭代：τt = τmax - (τmax - τmin) × (t / Ttotal)
  从 τmax（宽松，如基线参数的 100%）退火到 τmin（严格，如基线参数的 40%）

最终奖励：
  R(arch; t) = (-NMSEavg) × F2(Params(arch); τt, δ)

✅ 优势

  • 单次搜索即可获得完整 Pareto 前沿（vs. MnasNet 需要多次运行）
  • 策略平滑适应资源约束变化，避免过早收敛到局部最优
  • NPG-NAS 在 CIFAR-10/100 → ImageNet 上验证，仅需 ~7 GPU-day 即可完成全前沿搜索

1.4 方案三：Pareto 支配信用（ADC, NPG-NAS 风格）

来源：同上（NPG-NAS 的第二组件）

核心思想：维护一个动态 Pareto 前沿存档；新采样的架构若支配存档中的某些解，则获得正奖励；若被支配则获得负奖励。这直接优化 Pareto 支配关系，而非依赖加权标量公式。

Pareto 支配判定：
架构 A 支配架构 B（A ≻ B） ⇔
  (-NMSE)A ≥ (-NMSE)B  且  ParamsA ≤ ParamsB
  且至少一个不等式严格成立。

信用分配：
  若 A 支配 m 个存档解 → 正奖励 +α × m
  若 A 被 n 个存档解支配 → 负奖励 -β × n
  额外加入密度惩罚，避免前沿上某区域过于拥挤

✅ 优势

  • 不依赖加权标量，能到达非凸 Pareto 前沿上的所有解
  • 自动维护解的多样性（通过密度惩罚）
  • NPG-NAS 中 ADF + ADC 联合使用效果最佳

1.5 方案四：多层级奖励分解

为缓解延迟奖励（sparse reward）问题，可将单一的整体奖励分解为多个中间信号：

层级奖励信号时机
架构级 基于零成本代理（如 activation rank、SNIP）的即时评分 架构生成后立即获得（无需训练）
短训练级 300-epoch 快速训练的性能估计 架构生成后 ~6 分钟
完整训练级 1000-epoch 完整训练 + 多 case 验证 仅对进入 Pareto 前沿的候选架构触发

1.6 奖励函数方案对比总结

方案Pareto 覆盖实现复杂度超参敏感度推荐场景
加权乘积（MnasNet） 单点 ⭐ 低 高（w 需仔细调节） 快速原型验证、基线对比
退火期望函数 ADF 全前沿 ⭐⭐ 中 中（τ 范围需设定） 主要搜索运行
Pareto 支配信用 ADC 全前沿 ⭐⭐⭐ 高 中（密度惩罚系数） 与 ADF 联合使用
多层级分解 全前沿 ⭐⭐ 中 低 计算预算充足时的精细搜索

🎯 最终推荐

主方案：ADF + ADC 联合（NPG-NAS 路线），先在宽松资源约束下找到高性能区域，再逐步收紧资源约束，在单次搜索中覆盖完整 Pareto 前沿。

备用方案：加权乘积（MnasNet 路线），如果 ADF 调参困难，回退到简单的两目标加权乘积 + 多次不同权重运行。

性能指标选择：NMSE 为主指标（连续、稳定），EVM 和 ACLR 作为辅助约束（超过门限施加惩罚）。

第二部分：搜索时间过长的解决方案

2.1 问题量化

参数数值
单 case 完整训练（1000 epochs）≈ 20 分钟
单 case 短训练（300 epochs）≈ 6 分钟
每次评估使用 K 个 caseK = 3~5
单次完整评估（K=3，1000 epoch）≈ 1 小时
单次短评估（K=3，300 epoch）≈ 18 分钟
PPO 搜索需要的架构评估次数约 200~500 次
完整评估搜索总时间≈ 200~500 小时（8~21 天）
短评估搜索总时间≈ 60~150 小时（2.5~6 天）

即使是短训练方案，2.5~6 天的搜索时间仍然很长。需要进一步加速。

2.2 解决方案一：短训练代理 + 渐进淘汰（RANK-NOSH 路线）首选

来源：Wang et al., "RANK-NOSH: Efficient Predictor-Based Architecture Search via Non-Uniform Successive Halving", ICCV 2021

核心机制：非均匀连续折半（Non-uniform Successive Halving, NOSH）

多级淘汰调度示例：
阶段 1：对所有 N 个候选 → 训练 50 epochs → 按性能排序 → 淘汰后 50%
阶段 2：对剩余 N/2 个候选 → 训练 100 epochs → 按性能排序 → 淘汰后 50%
阶段 3：对剩余 N/4 个候选 → 训练 200 epochs → 按性能排序 → 淘汰后 50%
阶段 4：对剩余 N/8 个候选 → 完整训练 1000 epochs → 选出最优

时间节省：
相比所有候选都完整训练（N × 1000 epochs），NOSH 的总训练 epoch 数减少 约 60~70%。

✅ 为什么这比简单统一缩短到 300 epochs 更好？

  • 简单缩短到 300 epoch 对所有候选一视同仁，可能淘汰在早期表现一般但后期潜力大的架构
  • NOSH 给表现好的架构更多训练预算来"证明"自己，差的快速淘汰
  • RANK-NOSH 在 NAS-Bench-201 上以 15% 的搜索预算达到接近完整搜索的排名相关性（91.56% vs 91.60%）

2.3 解决方案二：权重共享超网（ENAS 路线）

来源：Pham et al., "Efficient Neural Architecture Search via Parameter Sharing", ICML 2018

核心思想

将所有候选架构嵌入一个过参数化的超网（SuperNet）中。每个候选架构是超网的一个子图，所有子图共享权重。评估候选架构时，直接从超网中提取对应子图的权重进行推理，无需独立训练。

步骤操作
1. 构建超网为每层包含所有可能的操作选择（如各种卷积核大小、卷积类型），构成一个大的计算图
2. 训练超网在多个 case 上训练超网至收敛（一次性开销，约数小时）
3. 采样评估从超网中采样子架构，直接使用共享权重评估性能（无需额外训练）
4. PPO 搜索使用采样评估结果作为奖励信号，驱动 PPO 搜索
5. 最终验证对搜索到的最优架构从头训练，获得真实性能

⚠️ 关键风险：优化差距（Optimization Gap）

共享权重下评估的架构排名可能与独立训练的真实排名存在偏差。Xie et al. (ACM Computing Surveys, 2021) 指出这是权重共享 NAS 的核心挑战。

缓解措施：

  • 超网训练时加入 FairNAS 的公平采样策略，确保每个操作被均匀训练
  • 搜索结束后对 Top-K 架构进行独立训练的"校准"步骤
  • 结合用户提出的"逐层复用不变层权重"思路 —— 这本质上是超网共享的一种"增量"变体

2.4 解决方案三：零成本代理预筛选（Zero-Cost Proxy）

来源：NEAR (2024), TSE (NeurIPS 2021), PATNAS (2024)

核心思想

在完全不训练的情况下，通过分析网络在随机初始化状态下的某些统计特性来预测其最终性能。以此作为"预筛选"工具，在进入昂贵训练评估前过滤掉明显劣质的架构。

代理指标原理计算时间
Activation Rank (NEAR, 2024) 计算前激活和激活后矩阵的有效秩，高有效秩暗示更强的表达能力 秒级
SNIP 基于连接敏感度的评分：|梯度 × 权重| 在单个 mini-batch 上的求和 秒级
SynFlow 数据无关的信号传播分析，评估网络避免梯度消失/爆炸的能力 秒级
TSE-EMA (NeurIPS 2021) 用近期训练 loss 的指数移动平均作为性能预测器，只需几个 epoch 分钟级

✅ 在本项目中的应用建议

零成本代理用于搜索空间粗筛：在 PPO 每轮采样后，先用 SNIP 或 Activation Rank 快速打分，只对得分前 50% 的候选架构进入 300-epoch 短训练评估。这相当于在方案一的基础上再加一层"免费"的过滤器。

2.5 三种加速方案的组合策略

🔹 第 0 层：零成本代理预筛

PPO 每轮采样 20 个架构 → SNIP 快速打分 → 保留前 10 个

时间：秒级 / 几乎免费

🔹 第 1 层：NOSH 渐进淘汰

10 个候选 → 50 epoch 淘汰 5 个 → 150 epoch 淘汰 3 个 → 保留 2 个进入完整训练

时间节省：~50%

🔹 第 2 层：逐层权重复用

对于架构中未改变的层，直接复用上一轮训练的权重，只训练新增/变更的层

时间节省：~20~30%

🔹 第 3 层：多 Case 并行

K 个 case 的训练并行执行（多 GPU 或数据并行），单次评估时间从 K×20 分钟降为约 20 分钟

时间节省：~K 倍

3.2 文献关键发现总结

  1. 加权标量奖励有根本性局限（TabNAS 2022）：在资源约束搜索空间中，MnasNet 风格奖励函数仅有 <1% 成功找到最优架构。推荐使用 NPG-NAS 的非平稳策略梯度（ADF + ADC）。
  2. 短训练排名相关性在 10-12 epochs 即达 0.6+（TSE 2021, RANK-NOSH 2021）：300 epochs 足以提供可靠的相对排序，远高于"安全线"。
  3. 零成本代理日趋成熟（NEAR 2024, PATNAS 2024）：Activation Rank 等指标在完全不训练的情况下即可提供有意义的性能预测，适合粗筛。
  4. PPO 用于 NAS + 量化联合搜索已有验证（MARCO 2024）：为本项目的两阶段方案（软参 + 量化）提供了直接的可行性支撑。
  5. 权重共享是最高效但最具风险的加速手段：ENAS 提供 1000× 加速，但"优化差距"问题需要额外的校准步骤（FairNAS, 2021）来缓解。

第四部分：综合实施建议

4.1 推荐技术路线图

阶段做什么验证指标
Step 0：验证短训练代理 随机采样 30~50 个架构，分别进行 300-epoch 和 1000-epoch 训练，计算 Spearman 排名相关性 ρ ≥ 0.7 即可使用 300-epoch 代理
Step 1：验证零成本代理 在相同 30~50 个架构上计算 SNIP / Activation Rank，与完整训练性能计算相关性 ρ ≥ 0.4 即可用于粗筛
Step 2：PPO 框架搭建 实现 LSTM 控制器 + ADF 奖励函数 + 300-epoch 短训练评估 + NOSH 淘汰 策略收敛曲线、搜索到架构的性能分布
Step 3：正式搜索运行 运行 PPO-NAS，目标 200~300 次有效评估，Pareto 前沿分析 前沿上选 3~5 点 vs 基线对比
Step 4：第二阶段探索 在选定架构上，PPO 搜索逐层量化位宽 存储 vs 性能的 Pareto 前沿

4.2 关键超参数建议

超参数建议值说明
PPO clip ε0.2标准值，控制策略更新幅度
每轮采样数8~16 个架构平衡样本多样性与评估时间
ADF 退火范围 τ[基线Params×40%, 基线Params×100%]从宽松到严格
NOSH 淘汰比例每阶段淘汰 50%3 阶段：50→150→1000 epochs
短训练 epoch 数300（主线）/ 50、150（NOSH 中间阶段）300 作为最终代理，50/150 用于快速淘汰
多 case 评估数 K3平衡泛化性与时间

4.3 风险与缓解

风险概率缓解措施
300-epoch 排名与 1000-epoch 排名相关性不足 中 Step 0 先验证；若不达标，提升至 500 epoch 或使用 TSE-EMA 作为补充信号
ADF 退火参数难以调优 中 准备回退方案：简单加权乘积 + 多次不同权重运行
搜索空间过大导致不收敛 低 先用离散化缩小空间（如核大小从 {3,5,7,9,11} 缩减为 {3,7,11}）
权重复用导致次优收敛 中 对 Top-K 候选始终进行从头训练的"去偏"评估
