③ 奖励计算

libs/reward.py:19 — compute_reward()
def compute_reward(nmse, num_params, config):
    perf_score = -nmse * acc_weight         # 性能: NMSE越小→得分越高
    param_penalty = param_weight * (num_params / target_params)
    return perf_score - param_penalty    # R 越大越好

④ GAE 优势计算

    δ = r + γ·V(s_{t+1}) - V(s_t)  # TD error
    A_t = δ + γ·λ·A_{t+1}          # 指数加权累积
A = (A - mean) / std                # 归一化
R_target = A + V(s)                 # Critic 训练目标

3. Controller：Actor-Critic LSTM 详解

📄 libs/model/controller.py [303 行]
🏛️ 网络结构： 
输入 (21维 one-hot 特征)
       │
       ▼
  input_proj: Linear(21 → 128)
       │
       ▼
  共享 LSTM (2层, hidden=128)
       │
       ├──────────────────────────────────────┐
       ▼                                      ▼
  Actor 头 (4个独立 Linear)          Critic 头
  ├─ actor_conv:   128→2  (卷积类型)      Linear(128→128) + ReLU
  ├─ actor_kernel: 128→5  (核大小)        Linear(128→64)  + ReLU
  ├─ actor_init:   128→3  (初始化策略)    Linear(64→1)    → V(s)
  └─ actor_spline: 128→5  (样条数,LUT用)

⏱️ 时间步展开（7步）

t=0: start_token → LSTM → h₀
     ├─ Actor 采样: conv, kernel, init
     └─ Critic: V(s₀)

t=1: encode(arch[0]) 21维 → LSTM → h₁
     ├─ Actor 采样: conv, kernel, init
     └─ Critic: V(s₁)

...

t=3 (LUT层): 只采样 spline (1个分布)

...

t=6: encode(arch[5]) → LSTM → h₆
     ├─ Actor 采样: conv, kernel, init
     └─ Critic: V(s₆)

6. PPO 更新（Loss 计算）

📄 libs/ppo.py :: _ppo_update() [line 153]
🧮 一次 PPO epoch 内的计算图： 
/* 输入 (全部 detach, 当作常量标签) */
old_log_probs (B,7)   ← rollout 时的策略概率
old_values    (B,7)   ← rollout 时的 Critic 估值
advantages    (B,7)   ← GAE 计算的优势 (归一化后)
returns       (B,7)   ← GAE 计算的回报 = A + V

/* 当前策略重新推理 */
new_log_probs, new_values, new_entropies = controller.evaluate(archs, actions)
                                          ↑
                                    只有这三个是"活"的（连着梯度）

/* ── Actor Loss ── */
ratio = exp(new_log_probs − old_log_probs)        // (B,7) 策略变化比
surr1 = ratio × advantages                        // 不加约束的目标
surr2 = clamp(ratio, 0.8, 1.2) × advantages      // clip 约束的目标
actor_loss = −mean( min(surr1, surr2) )           // 取 min → 安全阀

/* ── Critic Loss ── */
critic_loss = MSE(new_values, returns)             // 价值回归

/* ── 总损失 ── */
loss = actor_loss + 0.5×critic_loss − 0.01×mean(new_entropies)
       ↑                ↑                     ↑
  策略优化         价值估计              鼓励探索（最大化熵）

/* ── 梯度更新 ── */
optimizer.zero_grad()
loss.backward()                    // 梯度流过 new_log_probs, new_values
clip_grad_norm(max=0.5)           // 梯度裁剪
optimizer.step()                   // 更新 Controller 所有参数

🔒 Clip 机制详解

# 当 advantage > 0 (好动作)
ratio = 1.3  →  min(1.3·A, 1.2·A) = 1.2·A  # 被 clip
ratio = 1.1  →  min(1.1·A, 1.1·A) = 1.1·A  # 正常通过

# 当 advantage < 0 (坏动作)
ratio = 0.6  →  max(0.6·A, 0.8·A) = 0.8·A  # 被 clip (min取更负的)
ratio = 0.9  →  max(0.9·A, 0.9·A) = 0.9·A  # 正常通过

clip 范围内的样本 → 梯度正常回传
clip 范围外的样本 → ratio 对梯度贡献为 0（被 min 截断）

🔀 Minibatch 打乱

for epoch in range(4):           # 每个 epoch:
    indices = randperm(16)       # 重新随机排列
    for start in range(0, 16, 4):  # 分4个 minibatch
        idx = indices[start:start+4]
        # ... 用这4个样本算 loss → backward

同一批 16 个架构 × 4 epochs = 16 次梯度更新
每次 minibatch=4，每次打乱顺序不同

7. GAE 优势计算详解

📄 libs/ppo.py :: _compute_gae() [line 113]
⏮️ 从后往前累积（NAS 特化版）： 
/* 假设: B=16, T=7, 只有最后一步 t=6 有奖励 */

t=6 (最后一层):  δ₆ = R_final − V(s₆)                    // 直接拿奖励减价值
                A₆ = δ₆                                  // 最后一步，无后续

t=5:            δ₅ = γ·V(s₆) − V(s₅)                    // 无奖励，纯价值差
                A₅ = δ₅ + γ·λ·A₆                        // 把 t=6 的优势传过来

t=4:            δ₄ = γ·V(s₅) − V(s₄)
                A₄ = δ₄ + γ·λ·A₅

...以此类推...

A = (A − mean(A)) / std(A)      // 标准化到零均值单位方差
returns = A + V(s)               // Critic 训练目标

/* 直观理解: 找到好架构 → 所有参与决策的步骤都获得正优势
             找到差架构 → 所有决策步骤都获得负优势
             但每步的"贡献度"不同 (靠近最终决策的步信号更强) */
