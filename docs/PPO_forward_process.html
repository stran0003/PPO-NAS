2. 阶段一：采样 (forward)

2.1 调用入口

2.2 LSTM 时间步展开 (7步)

t=0 (L1线性):  start_token → LSTM → h₀
  ├─ actor_conv(h₀):   Linear(128→2) → logits → softmax → P(conv) → sample → c₀
  │     log_prob += log P(c₀)            # 始终计入
  ├─ actor_kernel(h₀): Linear(128→5) → logits → softmax → P(kernel) → sample → k₀
  │     log_prob += log P(k₀) × mask   # 仅 grouped 时计入, 1x1→k₀=-1
  ├─ actor_init(h₀):   Linear(128→? ) → logits → softmax → P(init) → sample → i₀
  │     log_prob += log P(i₀)            # 始终计入
  └─ critic(h₀):       MLP(128→128→64→1) → V(s₀)

  step_entropy = H(P_conv) + H(P_kernel)×mask + H(P_init)
  step_log_prob = 上述三项log_prob之和

t=1 (L2线性):  encode(arch[0], 21维) → LSTM → h₁ → 同上 ...

t=3 (L4 LUT):  只采样 spline (1个分布)
  step_log_prob = log P(spline)
  step_entropy  = H(P_spline)

t=4~6: 同 t=1~3 ...

2.3 输出张量

变量维度含义

log_probs(16, 7)每步采样动作的 log 概率 (pi_old)

3.2 奖励函数设计 ⭐

代码: libs/reward.py:19-57

当前策略：简单线性加权 (type: simple)

R = −nmse × acc_weight − param_weight × (num_params / target_params)
参数默认值含义
acc_weight1.0性能权重 — NMSE越好奖励越高
param_weight0.1参数量惩罚 — 超过目标扣分
target_params100000目标参数量上限

数值举例

架构A: nmse=-32, params=120000
  perf_score = -(-32) × 1.0 = 32.0
  param_ratio = 120000 / 100000 = 1.2
  param_penalty = 0.1 × 1.2 = 0.12
  R_A = 32.0 − 0.12 = 31.88  ✅ 高分

架构B: nmse=-28, params=80000
  R_B = 28.0 − 0.08 = 27.92  ❌ 性能差

架构C: nmse=-32, params=200000
  R_C = 32.0 − 0.20 = 31.80  ⚠️ 性能好但太大
设计要点
• NMSE取负号后变成正数，性能越好分数越高
• 延迟奖励：整个架构生成完毕才给一次奖励，中间步骤 reward=0
• 奖励是PPO的唯一学习信号：Controller完全靠这个标量知道"什么样的架构好"

4. 阶段三：GAE 优势计算

4.1 为什么需要 GAE

NAS是序列决策——Controller分7步生成架构，但奖励只在最后一步给。GAE将终端奖励沿时间轴反向回传，让每一步都知道自己的贡献。

4.2 计算过程 (NAS特化版)

代码: libs/ppo.py:116-152

for t in reversed(range(7)):           # t=6,5,4,3,2,1,0
    if t == 6:                           # 最后一步: 直接拿奖励
        δ = reward − V(s₆)
        A₆ = δ
    else:                                # 中间步骤: 奖励=0, 纯价值差
        δ = 0.99 × V(s_{t+1}) − V(s_t)
        A_t = δ + 0.99×0.95 × A_{t+1}

# 后处理
returns = A + V(s)                     # Critic 训练目标
A = (A − mean(A)) / (std(A) + 1e-8)   # 零均值单位方差归一化

4.3 直观理解

时间步V(s)rδA (累积)含义
t=63031.88+1.88+1.88最后一步贡献了正的
t=52900.99×30−29=0.70.7+0.94×1.88=2.47继承了t=6的正优势
t=42800.99×29−28=0.710.71+0.94×2.47=3.03优势继续向前传递

A_t > 0 → 这一步比预期好 → 增加概率；A_t < 0 → 这一步比预期差 → 降低概率

标准化后 advantages 均值=0，这就是Actor loss接近零的数学原因。全部detach后作为固定标签传入PPO更新。

5. 阶段四：PPO 更新循环

代码: libs/ppo.py:156-230 (_ppo_update)

5.1 循环结构

epoch 1: randperm(16) → [3,12,0,7, 9,2,14,5, 1,8,11,15, 6,10,4,13]
         minibatch[0]=[3,12,0,7]   → evaluate() → loss → backward() → step()
         minibatch[1]=[9,2,14,5]   → evaluate() → loss → backward() → step()
         minibatch[2]=[1,8,11,15]  → ...
         minibatch[3]=[6,10,4,13]  → ...

epoch 2: randperm(16) → [14,1,9,6,...]  ← 重新打乱, 不同组合
         ...

每 epoch 重新打乱 → 16个样本以不同组合出现在minibatch中 → 更充分的梯度利用

5.2 重新评估 evaluate() — 用新策略看旧动作

# controller.evaluate(batch_archs, batch_actions) 内部:
for t in range(7):
    h_t = LSTM(encode(arch[t-1]))      # 用当前参数重新前向
    logits = actor_conv(h_t)           # 当前Actor头输出
    dist = Categorical(logits=logits)
    lp = dist.log_prob(action_old[t])  # 用旧动作查新分布的概率 ← pi_new!
    ent = dist.entropy()               # 当前分布的熵
    V = critic(h_t)                    # 当前Critic估值

5.3 三项 Loss 详解

Actor Loss — Clipped Surrogate

ratio = exp(new_log_probs − old_log_probs)          # (4,7)
surr1 = ratio × advantages                           # 不加约束
surr2 = clamp(ratio, 0.8, 1.2) × advantages           # clip约束
actor_loss = −mean( min(surr1, surr2) )               # 取min = 安全阀
advantageratiomin选谁效果
+ (好动作)ratio>1 (更倾向)surr2 (clip截断)不过度增加概率
+ (好动作)ratio<1 (变冷淡)surr1 (正常)惩罚变冷淡
− (坏动作)ratio<1 (变冷淡)surr2 (clip截断)不过度降低概率
− (坏动作)ratio>1 (更倾向)surr1 (正常)惩罚更倾向

Critic Loss — MSE价值回归

critic_loss = MSE(new_values, returns)    # returns = A_GAE + V_old

熵正则化 — 鼓励探索

entropy = new_entropies.mean()            # (4,7) → 标量

总损失

loss = actor_loss + 0.5 × critic_loss − 0.01 × entropy

5.4 梯度更新

optimizer.zero_grad()
loss.backward()
clip_grad_norm_(max=0.5)      # 梯度裁剪, 防止爆炸
optimizer.step()

6. 阶段五：反向传播与梯度流

代码: libs/ppo.py:214-217

6.1 各项 Loss 对参数的梯度贡献

Loss 项LSTM + input_projActor头Critic头
actor_loss ✅ (通过h_t) ✅ (直接) ❌
critic_loss ✅ (通过h_t) ❌ ✅ (直接)
entropy (−) ✅ (通过h_t) ✅ (直接) ❌
总计 ✅✅✅ (三者之和) ✅✅ (actor+entropy) ✅ (仅critic)

6.2 逐项梯度流详解

Actor Loss 的梯度流

actor_loss
    │
    ▼
ratio = exp(new−old)
    │
    ▼ gradient
actor_conv / actor_kernel
actor_init / actor_spline
    │
    ▼ gradient
   h_t → LSTM → input_proj

❌ Critic头 无梯度

Critic Loss 的梯度流

critic_loss = MSE(V,ret)
    │
    ▼ gradient
critic (128→128→64→1)
    │
    ▼ gradient
   h_t → LSTM → input_proj

❌ Actor头 无梯度

Entropy 的梯度流

entropy = mean(H(π))
    │
    ├──→ actor_conv
    ├──→ actor_kernel
    ├──→ actor_init
    └──→ actor_spline
    │
    ▼ gradient
   h_t → LSTM → input_proj

❌ Critic头 无梯度

6.3 第一轮 epoch 的特殊情况

epoch 1: ratio = 1.0 (θ_new = θ_old 尚未更新)
advantages 零均值 → actor_loss ≈ 0 → Actor loss 不贡献有效梯度
但: critic_loss ≠ 0 (Critic随机初始化) + entropy ≠ 0
→ LSTM被 critic_loss 和 entropy 更新 → h_t 变了

epoch 2: h_t 变化 → new_log_probs ≠ old_log_probs → ratio ≠ 1.0
→ actor_loss 开始贡献梯度 → Actor头被更新

结论: 第一轮不浪费 — Critic和Entropy帮LSTM"热身"，为Actor的后续优化铺路
