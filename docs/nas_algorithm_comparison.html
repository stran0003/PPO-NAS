1. PPO 算法详解（当前方案）

1.1 算法架构

PPO 是一种 策略梯度 强化学习方法，使用 Actor-Critic 双网络结构。当前代码采用 LSTM 控制器同时输出 Actor（动作概率）和 Critic（状态价值）。

核心组件

组件作用
Actor输出每个架构设计决策的概率分布（选什么卷积、什么核大小……）
Critic估计当前状态的价值 V(s)，用于 GAE 计算优势
GAE将终端奖励沿时间步回传，计算每步的优势函数
Clip限制策略更新幅度，防止训练崩溃

关键超参数

参数值
rollouts_per_iter16
ppo_epochs4
clip_epsilon0.2
gamma0.99
gae_lambda0.95
entropy_coef0.01

1.2 Actor Loss：Clipped Surrogate Objective

Actor Loss = −𝔼[ min( rₜ(θ) · Âₜ , clip(rₜ(θ), 1−ε, 1+ε) · Âₜ ) ]

其中：

  • rₜ(θ) = π_new(a|s) / π_old(a|s) — 新旧策略的概率比，衡量策略变化程度
  • Âₜ — GAE 计算的归一化优势（见 1.4 节）
  • ε = 0.2 — clip 范围 [0.8, 1.2]
直觉：当优势为正（好动作），不让 ratio 超过 1.2（防止过度增加该动作概率）；当优势为负（坏动作），不让 ratio 低于 0.8（防止过度压制该动作概率）。取 min 保证了当 ratio 超出 clip 范围时梯度为 0，产生一个"安全阀"。

1.3 Critic Loss：MSE 价值回归

Critic Loss = MSE( V_new(s), R_target )

其中 R_target = A_GAE + V_old(s)（即 GAE 回报），是"这个状态实际值多少钱"的标签。Critic 学习准确预测它，让 GAE 的优势估计越来越准。

1.4 GAE 优势计算

δₜ = rt+1 + γ·V(st+1) − V(sₜ)   (TD-error)
Aₜ = Σl=0∞ (γλ)l · δt+l

在 NAS 中，中间步骤 reward=0，只有最后一步给奖励。GAE 通过 Critic 估计的 V(s) 将终端奖励 沿决策链回传，实现逐步骤的功劳分配。

1.5 总损失

Total Loss = Actor_Loss + 0.5 × Critic_Loss − 0.01 × Entropy

减去熵是为了 鼓励探索（熵越大策略越随机），防止过早收敛到局部最优。

1.6 PPO 的优缺点

✅ 优势

  • 训练稳定 — clip 机制防止策略突变
  • 样本效率好 — 同一批数据可复用多个 epoch
  • Critic 提供精细的时序功劳分配
  • 成熟可靠 — 工业界广泛验证
  • 适合 NAS 的序列决策结构

❌ 劣势

  • 需要训练 Critic 网络（额外的参数和计算）
  • 对超参数敏感（clip_epsilon, gamma, lambda）
  • 评估成本高 — 每个架构需要完整训练
  • 搜索效率低 — 500 轮 × 16 个架构 = 8000 次训练

2. GRPO 算法详解

2.1 来源与动机

GRPO 由 DeepSeekMath (Shao et al., 2024) 提出，因 DeepSeek-R1 (Guo et al., 2025) 使用它训练出强大推理能力而成为 2025 年最受关注的 RL 算法之一。

核心创新：干掉 Critic！ GRPO 不再学习价值函数 V(s)，而是通过组内相对比较（同一 prompt 生成多个回答，互相比较）来估计优势。这省掉了 Critic 网络的参数、内存和计算开销。

2.2 算法流程

  1. 组采样：对每个 prompt q，从旧策略生成 G 个回答（G=4~16）
  2. 奖励计算：用可验证奖励（数学题的答案对错、代码的通过率等）给每个回答打分
  3. 组归一化：Aᵢ = (rᵢ − mean(r₁..r_G)) / std(r₁..r_G) — 比组内平均好的得正优势，差的得负优势
  4. 策略更新：用 PPO 风格的 clip + KL 散度正则化更新 Actor

2.3 关键公式

优势: Aᵢ = (rᵢ − μ_group) / σ_group
损失: max_θ 𝔼[ min(w·A, clip(w, 1−ε, 1+ε)·A) − β·KL(π_θ || π_ref) ]

其中 β 是 KL 正则化系数，防止策略偏离参考模型太远。

2.4 GRPO 的优缺点

✅ 优势

  • 无 Critic — 内存/计算减半
  • 实现更简单 — 不需要 GAE、价值函数
  • 组归一化天然控制优势的方差
  • 在 LLM 推理任务上效果出众

❌ 劣势

  • 缺乏时序功劳分配 — NAS 场景致命问题
  • 同质组无学习信号 — 全对/全错时方差=0
  • 需要更大采样量 — G 个回答 vs PPO 的单个
  • 长时序任务中 Critic 更优（NeurIPS 2025 证实）

3. DARTS 算法详解

3.1 核心思想

DARTS（Differentiable ARchiTecture Search）由 Liu et al. (CMU/Google, ICLR 2019) 提出。它与 PPO/GRPO 的根本区别在于：

PPO/GRPO = 强化学习范式：把架构搜索当作序列决策问题，用奖励信号驱动搜索。
DARTS = 梯度优化范式：把离散的架构选择松弛为连续的 softmax 权重，直接用梯度下降优化架构参数。

结果：DARTS 搜索仅需 4 GPU 天，而 RL-based NAS 需要 2000+ GPU 天 —— 三个数量级的加速。

3.2 连续松弛（核心机制）

DARTS 搜索的是 计算单元（Cell），网络由这些 Cell 堆叠而成。每个 Cell 是一个有向无环图（DAG）。

连续松弛: o̅(i,j)(x) = Σo∈O softmax(αo(i,j)) · o(x)

每条边不再选择一个操作，而是所有候选操作的加权混合。权重由架构参数 α 的 softmax 决定。搜索结束后取 argmax 得到离散架构。

3.3 双层优化

上层: minα Lval(w*(α), α)   — 优化架构参数 α
下层: w*(α) = argminw Ltrain(w, α)   — 优化网络权重 w

交替更新 α 和 w，用一步近似避免每次 α 更新都要完全训练 w。

3.4 DARTS 的优缺点

✅ 优势

  • 极快 — 搜索只需 0.5~4 GPU 天（vs RL 的几百上千天）
  • 端到端可微分 — 直接梯度优化，无策略梯度方差问题
  • 收敛稳定 — 不依赖奖励函数设计
  • 同时搜索操作和拓扑

❌ 劣势

  • GPU 内存巨大 — 整个 DAG 同时驻留在显存中
  • Skip-connection 主导导致性能坍塌
  • 离散化差异 — 连续 softmax 与最终 argmax 之间有性能鸿沟
  • Cell 结构不够灵活 — 固定 DAG 拓扑，无法搜索全局架构
  • 搜索空间限定 — 只搜 Cell，不搜索层数、通道数等宏观结构

4.1 核心维度对比

范式 强化学习 强化学习（无Critic） 梯度优化

4.2 算法机制对比图

═══ PPO（强化学习 + Actor-Critic） ═══

  采样架构 → 训练评估 → 奖励 r
                              ↓
  Critic 估算 V(s) → GAE 计算优势 Aₜ
                              ↓
  Actor Loss = -min(rₜ·Aₜ, clip(rₜ)·Aₜ)
  Critic Loss = MSE(V(s), R_target)
                              ↓
  梯度更新 Actor + Critic（复用数据 4 epoch）


═══ GRPO（强化学习 + 无 Critic） ═══

  采样 G 个架构 → 各训练评估 → G 个奖励
                              ↓
  组归一化: Aᵢ = (rᵢ − mean) / std
                              ↓
  Actor Loss = -min(w·A, clip(w)·A) + β·KL(π||π_ref)
                              ↓
  梯度更新 Actor（无 Critic）


═══ DARTS（梯度优化 + 连续松弛） ═══

  所有操作的 softmax 加权混合 → 连续的前向传播
                              ↓
  ∇α L_val ← 验证集上反向传播梯度到架构参数 α
  ∇w L_train ← 训练集上反向传播梯度到网络权重 w
                              ↓
  交替更新 α 和 w（无需 RL 采样）
  搜索结束 → argmax α → 离散架构

4.3 对当前 DPD-NAS 场景的适用性评级

算法适用性评级理由
PPO ⭐⭐⭐⭐⭐ 最佳 当前方案 序列决策天生匹配逐层架构选择；GAE 提供精细功劳分配；clip 保证训练稳定
GRPO ⭐⭐⭐ 可用但需改造 备选 需要解决"序列级奖励如何分配到每层决策"的问题；可省掉 Critic 但时序信息丢失
DARTS ⭐⭐ 不适合 不推荐 DPD 模型有固定的 7 层顺序结构（不是 Cell 堆叠）；每层异构（3线性+LUT+3线性），DARTS 的 DAG 拓扑难以表达；内存需求巨大

5. 适用场景建议

🏗️ PPO

序列决策架构搜索

逐层选择、异构结构
可变长度、灵活搜索空间
需要精细 Credit Assignment

如：DPD 混合模型、CNN 逐层搜索、RNN 结构搜索

💬 GRPO

LLM 推理对齐

数学推理、代码生成
有可验证奖励的场景
大模型训练（省显存）

如：DeepSeek-R1、数学解题、代码竞赛

🔬 DARTS

Cell 级图像架构搜索

CNN Cell 结构搜索
固定宏架构 + 搜索微结构
算力受限但追求速度

如：CIFAR/ImageNet 分类网络、轻量级移动端模型
